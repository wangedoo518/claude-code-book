## 这本书写给谁

**写给每一个好奇"AI 编程助手到底是怎么工作的"的人。**

- 如果你是**高中生或编程初学者**——每章开头的"生活类比"和"先回答一个问题"就是为你准备的。你不需要读懂每一行源码，但你会理解一个真实的大型软件是怎么设计的。
- 如果你是**有经验的开发者**——"源码拆解"和"设计取舍"部分会带你深入调用链和架构决策。
- 如果你是**架构师或 AI Agent 开发者**——每章的"深水区"和全书的设计思想主线就是为你准备的。

**一本书，三种读法，同一条路径。**

## 全书四条设计思想主线

> 🧭 **主线一**：Claude Code 不是聊天壳——它是一个"会调用工具的任务执行器"
>
> 🧭 **主线二**：核心铁三角——"上下文装配 + Agent Loop + 工具编排"
>
> 🧭 **主线三**：最难的不是生成代码——而是"权限、安全、压缩、恢复、一致性"
>
> 🧭 **主线四**：两套代码库——"能跑起来"不等于"官方原始设计"

## 与 ccbook 的关系

本书不是 ccbook 的"升级版"，而是**并行的另一条路径**：

| 维度 | ccbook《源码解读》 | 本书《红宝书》 |
|------|-------------------|---------------|
| 读者 | 高中生为主 | **高中生到架构师全覆盖** |
| 叙事 | 比喻为主 | **类比开门 + 源码验证 + 设计推演** |
| 深度 | 概念入门 | **渐进式：表层→核心→深水区** |
| 代码 | 简化伪代码 | **简化图 + 真实源码双轨呈现** |
| 双生库 | 未区分 | **每章区分还原层/补全层** |
| 安全 | 3章概述 | **5章 + 25道关卡** |
| 隐藏 | 未涉及 | **KAIROS/Flag/彩蛋** |
| 竞品 | 未涉及 | **12维度 + 生态全景** |
| 阅读指导 | 顺序阅读 | **三种读法 + 推荐路径** |
